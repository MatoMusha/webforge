Three agents, one prompt

No slash commands. No configuration. Just say what you want and the agents coordinate automatically.

1

Director scans your project

Detects your framework, existing tokens, components, and conventions. Figures out if you have a design system or need one.

2

Strategist interviews you

If no design system exists, asks about purpose, audience, mood, color preferences, and typography feel. Generates a complete OKLCH token system.

3

Builder creates production code

Uses your tokens (new or existing) to build components and pages. Adapts to your stack — React, Vue, Svelte, or vanilla HTML.

Meet the agents

Director

Design System

What it builds for you

From a single prompt to a complete, production-ready foundation.

OKLCH Color Palette

Perceptually uniform colors with tinted neutrals, semantic tokens, and dark mode support.

Typography System

Distinctive fonts, fluid type scale with clamp(), proper line-heights, and font loading strategy.

Spacing & Layout

4pt grid, semantic spacing tokens, container queries, and responsive grid patterns.

Motion Tokens

Duration tiers, exponential easing curves, and prefers-reduced-motion support baked in.

Production Components

Semantic HTML, all 8 interaction states, keyboard navigation, WCAG AA contrast.

Stack Adaptation

Outputs Tailwind config, CSS custom properties, or both — matches your existing project.
